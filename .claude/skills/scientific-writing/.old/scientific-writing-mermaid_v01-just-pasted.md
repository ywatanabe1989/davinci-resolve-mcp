✅ Mermaid Academic Figure Style Guide (for Scientific/Engineering Diagrams)
1. General Principles

Clarity over decoration

Avoid gradients, shadows, rounded corners unless necessary

Keep node shapes consistent within a diagram

Prefer simple rectangular or hexagonal nodes for clarity

Semantic coloring (using your RGB palette)
Assign colors by function, not aesthetics:

Modules / components → blue, navy, lightblue

Data stores → yellow, orange

AI / computation → red, purple

External integrations → green

Neutral / glue systems → gray

Directional flow

Prefer top → bottom (conceptual → implementation)

Or left → right for pipelines

Avoid crossing edges when possible

Edge labeling

Label edges only when needed

Keep labels short (e.g., “API”, “SSH”, “SLURM job”)

Typography

Use concise component names

For academic diagrams:
No emojis, no decorative characters

Node spacing

Provide enough spacing (::: class) to avoid visual clutter

3. Recommended Node Shapes

Use these shapes consistently:

✔ System Modules
rect --> for Writer, Code, Vis, Scholar, Files

✔ AI / Compute Layers
hexagon --> AI Engine, LLM, Scheduler

✔ Databases / Storage
cylinders --> PostgreSQL, Redis, TempStorage

✔ External Services / API
rounded rect --> Cloudflare, Nginx, SSH Gateway

4. Example: Academic-Style Architecture Diagram (with Your Style)
flowchart TB

    %% Nodes
    AI[AI Engine]:::ai

    Writer[Writer]:::module
    Files[Files]:::module
    Code[Code]:::module
    Vis[Vis]:::module
    Scholar[Scholar]:::module

    %% Edges
    AI --> Writer
    AI --> Code
    AI --> Vis
    AI --> Scholar

    Writer --> Files
    Code --> Files
    Scholar --> Files

    Files --> Writer
    Files --> Vis
    Files --> Code

    Code --> Vis
    Vis --> Writer

    %% Class Assignments
    class AI ai;
    class Writer,Files,Code,Vis,Scholar module;


This produces a clean, research-paper-grade diagram, with a consistent palette.

6. Guidelines for Scientific Readability
✔ Show layers

Presentation layer

Application layer

Service layer

Storage layer

Compute layer

✔ Show direction

Use arrows consistently

Top → bottom for conceptual → implementation

Left → right for workflows

✔ Reduce color noise

Use at most 4–5 colors in one diagram.

✔ Avoid long node names

If needed, place details in a footnote, not the box.

✔ Keep edges straight

Minimize curved edges unless visual clarity demands it.
